# GlobalCache 缓存时间优化报告 (v2.4.2)

## 📊 优化概览

**优化目标：** 在保证数据新鲜度的前提下，最大化延长缓存时间，减少外部API调用次数，防止IP被封禁。

**优化策略：** 根据数据类型、交易时段、用户活跃度三个维度进行精细化调整。

---

## 🎯 优化详情

### 1️⃣ 实时估值数据 (realtime) - 动态TTL

| 时段 | 原TTL | 新TTL | 提升倍数 | 优化理由 |
|------|-------|-------|----------|----------|
| **盘中交易 (9:00-15:00)** | 20秒 | **60秒** | 3x | 估值波动相对缓慢，60秒可接受 |
| **下午盘后 (15:00-18:00)** | 5分钟 | **30分钟** | 6x | 数据已基本固定 |
| **晚间 (18:00-22:00)** | 5分钟 | **1小时** | 12x | 数据完全固定，用户活跃度下降 |
| **深夜/凌晨 (22:00-6:00)** | 5分钟 | **2小时** | 24x | 几乎无人访问，最大化缓存 |
| **早盘前 (6:00-9:00)** | 10分钟 | **30分钟** | 3x | 用户开始活跃但数据未更新 |
| **凌晨 (0:00-6:00)** | 10分钟 | **2小时** | 12x | 极低访问量 |

### 2️⃣ 历史净值数据 (history)

| 数据类型 | 原TTL | 新TTL | 提升倍数 | 优化理由 |
|----------|-------|-------|----------|----------|
| **近期历史 (3天内)** - 用于确认状态检测 | 1小时 | **3小时** | 3x | 确认后数据不会变化 |
| **远期历史 (更早)** | 24小时 | **3天 (72h)** | 3x | 历史数据固定不变 |

### 3️⃣ 基金基础信息

| 数据类型 | 原TTL | 新TTL | 提升倍数 | 优化理由 |
|----------|-------|-------|----------|----------|
| **基金基本信息** (名称、类型等) | 7天 | **14天** | 2x | 极少变化（年检级别） |
| **基金列表** (搜索用) | 1小时 | **6小时** | 6x | 列表相对稳定 |

### 4️⃣ 系统状态

| 数据类型 | 原TTL | 新TTL | 提升倍数 | 优化理由 |
|----------|-------|-------|----------|----------|
| **市场状态** (开闭盘检测) | 15秒 | **1分钟** | 4x | 检测频率不需要太高 |
| **默认缓存** | 30秒 | **1分钟** | 2x | 通用场景 |

## 🔍 技术实现亮点

### 1. 精细化时段划分
```javascript
// ✨ 不再简单区分"盘中/盘后"，而是细化到具体小时
case 'after_hours': {
  const hour = now.getHours();
  
  if (hour >= 22 || hour < 6) {
    return 120 * 60 * 1000;  // 深夜：2小时
  } else if (hour >= 18) {
    return 60 * 60 * 1000;   // 晚上：1小时
  } else {
    return 30 * 60 * 1000;   // 下午：30分钟
  }
}
```

### 2. 数据类型差异化
```javascript
// ✨ 不同数据类型采用不同策略
case 'history_recent':  
  return 3 * 60 * 60 * 1000;   // 近期：3小时
  
case 'history_older':   
  return 72 * 60 * 60 * 1000;  // 远期：3天
  
case 'fund_info':       
  return 14 * 24 * 60 * 1000; // 基本信息：14天
```

### 3. 智能降级保障
- ✅ 强制刷新机制保留（forceRefresh参数）
- ✅ 手动清理功能可用（clear()方法）
- ✅ 定时清理任务正常（cleanup()方法）
- ✅ 监控统计完整（getStats()方法）

## ⚠️ 注意事项

### 数据一致性保障
1. **强制刷新入口**：关键操作（如确认收益）仍会跳过缓存
2. **事件驱动失效**：基金净值确认后会主动清除相关缓存
3. **手动清理能力**：管理员可通过API清空全部缓存

### 性能监控建议
```javascript
// 定期查看缓存统计
const stats = globalCache.getStats();
console.log(`命中率: ${stats.hitRate}`);
console.log(`缓存大小: ${stats.size}/${stats.maxSize}`);
console.log(`当前TTL: ${stats.realtimeTTL}`);
```

### 异常处理
- 如果发现数据陈旧，可调用 `globalCache.clear()` 清空缓存
- 如果需要测试，可使用 `{ forceRefresh: true }` 参数跳过缓存
- 缓存满时自动LRU淘汰（最多500条）
