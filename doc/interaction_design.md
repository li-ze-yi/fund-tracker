### 2.2 组件详细规范

#### 2.2.1 基础原子规范

**间距网格 (Spacing Scale)**

| Token | 数值 | 使用场景 |
|-------|------|----------|
| --space-xxs | 4px | 图标与文字间距、标签内间距 |
| --space-xs | 8px | 组件内元素间距、列表项内间距 |
| --space-sm | 12px | 表单标签与输入框间距 |
| --space-md | 16px | 卡片内边距、列表项间距 |
| --space-lg | 20px | 区块间距、弹窗内边距 |
| --space-xl | 24px | 页面边距、大区块间距 |
| --space-2xl | 32px | 页面标题与内容间距 |
| --space-3xl | 48px | 大区块分隔间距 |

**圆角规范 (Border Radius)**

| Token | 数值 | 使用场景 |
|-------|------|----------|
| --radius-sm | 4px | 标签、小图标 |
| --radius-md | 8px | 卡片、输入框、按钮 |
| --radius-lg | 12px | 弹窗、大卡片 |
| --radius-xl | 16px | 底部面板 |
| --radius-full | 50% | 头像、圆形图标 |

**阴影层级 (Shadow Elevation)**

| Token | 数值 | 使用场景 |
|-------|------|----------|
| --shadow-sm | 0 1px 2px rgba(0,0,0,0.06) | 卡片默认态 |
| --shadow-md | 0 4px 12px rgba(0,0,0,0.08) | 弹窗、下拉面板 |
| --shadow-lg | 0 8px 24px rgba(0,0,0,0.12) | 模态弹窗 |
| --shadow-xl | 0 12px 40px rgba(0,0,0,0.16) | 顶部通知、特殊浮层 |

**字体排印 (Typography)**

| Token | 数值 | 字重 | 使用场景 |
|-------|------|------|----------|
| --text-caption | 11px | 400 | 辅助文字、标签 |
| --text-body-sm | 12px | 400/500 | 表格内容、列表副信息 |
| --text-body | 14px | 400/500 | 正文、列表项主信息 |
| --text-body-lg | 15px | 500 | 基金名称、重要信息 |
| --text-subtitle | 16px | 600 | 弹窗标题、卡片标题 |
| --text-title | 18px | 600 | 页面标题 |
| --text-heading | 20px | 700 | 大标题 |
| --text-display | 24px | 700 | 数字展示（估值、金额） |
| --text-display-lg | 32px | 700 | 大数字展示（总资产） |

**三步流程**：

**步骤一：上传**
- 黄色警告提示框："功能测试中，识别可能不准确，主要支持支付宝持仓截图"
- 拖拽上传区域（InboxOutlined 图标 + 提示文字）
- 支持 JPG/PNG，最大 10MB
- 上传后显示图片预览 + "重新选择"按钮 + "开始识别"按钮

**步骤二：识别中**
- Spin 加载动画
- 提示文字："正在识别图片中的持仓信息..."
- 副文字："OCR 识别可能需要 10-30 秒，请耐心等待"

**步骤三：结果确认**
- 统计栏：有效 X 条 / 无效 X 条 + 分组选择器
- 桌面端：Table 表格
  - 基金代码（只读文本）
  - 基金名称（Input + 联动搜索下拉）
  - 持仓金额（InputNumber）
  - 累计收益（InputNumber）
  - 状态（Tag：有效/无效）
  - 操作（删除按钮）
- 手机端：卡片列表
  - 折叠态：基金名称+代码 | 金额+收益
  - 展开态：编辑表单（代码只读、名称搜索、金额、收益）
  - 无效卡片红色边框
  - 右上角删除按钮
- 底部按钮：重新上传 | 确认导入（N条）
- 可展开查看原始OCR识别文本

## 第三部分：配色方案设计

### 3.1 设计原则

基金实时估值软件的配色需遵循以下原则：

| 原则 | 说明 |
|------|------|
| **金融可信度** | 色彩传递专业、可靠、安全的品牌感知 |
| **数据可读性** | 涨跌颜色符合用户心智模型（红涨绿跌），数据层次清晰 |
| **视觉舒适度** | 长时间使用的眼部舒适性，避免高饱和度过度刺激 |
| **信息层级** | 通过色彩明度、饱和度区分信息优先级 |
| **品牌辨识度** | 独特的色彩组合形成品牌记忆点 |
