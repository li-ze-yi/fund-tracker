## 七、风险与阻塞项

| 风险 | 影响阶段 | 说明 | 缓解措施 |
|------|----------|------|----------|
| 天天基金接口变更/下线 | Phase 1~5 | 核心数据源不可用 | 东方财富接口备用，接口监控告警 |
| 全市场基金数据量过大 | Phase 1 | 首次拉取耗时 | 分批拉取，异步处理 |
| React Native 图表库手势支持 | Phase 4 | 缩放交互实现复杂度 | Web端先行验证，App端选型时重点评估 |
| 数据库连接稳定性 | Phase 1~5 | 远程 MySQL 可能不稳定 | 本地开发库兜底 |
| 外部估值接口更新频率低 | Phase 2~5 | 天天基金gszzl约1-3分钟更新一次，盘中刷新可能看不到变化 | 前端显示估值更新时间，引导用户理解数据节奏 |
| 历史净值缓存未支持forceRefresh | Phase 2~5 | 盘中30分钟TTL可能导致确认状态延迟 | 待修复：forceRefresh时历史净值也应跳过缓存 |
| **ICP 未备案导致手机无法访问** | Phase 5 | 阿里云对未备案域名进行网络层拦截，手机浏览器请求无法到达服务器（电脑可能因缓存暂时可访问） | 尽早提交 ICP 备案（1-3周）；临时方案：境外服务器或 Cloudflare CDN |
| 净值确认时间不确定 | Phase 3 | A股基金净值确认时间通常18:00-20:00，极端情况可能延迟 | 21:00 重试机制 + 未确认时跳过不更新 next_run_date |
